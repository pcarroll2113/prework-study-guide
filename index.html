HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML elements are case sensitive.
  • HTML does not engage in programming "logic" but it is good for displaying and structure.
  • HTML is like the frame of a house that provides the structure that the house is built upon.
  • All HTML elements are essentially a box.
  • An element is a type of content on a webpage; text elements, image elements, header elements, list elements.
  • Elements can also be used to divide content on the page and to describe the content to the browsers and other developers; meta-related elements, content division elements, semantic elements.
  • HTML elements are composed of a closing and opening tag represented with angle brackets.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • You must link the CSS file to the HTML file to have the styles apply to the application.
  • CSS Rules are portrayed as: selector {Property: Value; Property: Value;} (the portion inside the {-} is the declaration block)
  • The selector defines the element or attributes to which the rules (or declarations) will apply.
  • Declarations contain two components: the CSS property we want to apply and the value of the property.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git commit -m "write commit message here": commits all staged work (It is important to write a brief, clear commit message so you know what each commit is for!)
  • git pull: pulls down any changes from your remote
  • git push: pushes your local changes up to your remote
  • git branch: shows you all your local branches and indicates which branch you are currently on
  • git merge name-of-branch: merges the specified branch into the branch you are currently on
  • git diff: shows you the changes in your unstaged code

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript is a programming language that allows users to interact with websites
  • JavaScript is an object-oriented programming (OOP) language
  • One of its core features is variables, which store data types as values
  • Control flow is the order in which a computer executes code in a Javascript file
  • Conditional statements can be used to interrupt control flow
  • Variables are containers that store values
  • JavaScript is case sensitive
  • In JS write a comment as /* Everything in between is a comment */
  • “Commenting out code” makes it inactive without deleting it, just in case you need it later
  • // This is a comment with no line breaks
  • Null is a special value in JavaScript that refers to the absence of a value
  • Conceptually, undefined indicates the absence of a value, while null indicates the absence of an object
  • NaN ("Not a Number") is the only value in JavaScript that is not equal to itself
  • JavaScript strings are immutable-which means that once a string is created, it is not possible to modify it
  • Use strings for textual data. When representing complex data, parse strings, and use the appropriate abstraction
  • A Symbol is a unique and immutable primitive value and may be used as the key of an Object property (In some programming languages, Symbols are called "atoms”)
  • The purpose of symbols is to create unique property keys that are guaranteed not to clash with keys from other code
  • In JavaScript, objects are the only mutable values
  • Functions are also objects with the additional capability of being callable
  • In JavaScript, to declare a variable, we use the var keyword
  • All strings are surrounded by quotation marks
  • Each line of JavaScript code ends with a semicolon
  • A method is a set of instructions that can be carried out by a computer
  • An array is a single variable that holds a group of related data
  • In JavaScript arrays are zero-indexed and sequential
  • A for loop contains three important statements; first: the starting point for the loop, second: the condition, final statement is what allows the array to iterate over each item(1. Keyword 'for' 2.three part conditional statement 3.code block)
  • Creating a function defines a block of code that performs a specific task and gives it a name
  • To execute a function we must call it in our code!
  • Since control flow dictates that JavaScript files are executed from top to bottom, order matters
