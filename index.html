Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML elements are case sensitive.
  • HTML does not engage in programming "logic" but it is good for displaying and structure.
  • HTML is like the frame of a house that provides the structure that the house is built upon.
  • All HTML elements are essentially a box.
  • An element is a type of content on a webpage; text elements, image elements, header elements, list elements.
  • Elements can also be used to divide content on the page and to describe the content to the browsers and other developers; meta-related elements, content division elements, semantic elements.
  • HTML elements are composed of a closing and opening tag represented with angle brackets.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • You must link the CSS file to the HTML file to have the styles apply to the application.
  • CSS Rules are portrayed as: selector {Property: Value; Property: Value;} (the portion inside the {-} is the declaration block)
  • The selector defines the element or attributes to which the rules (or declarations) will apply.
  • Declarations contain two components: the CSS property we want to apply and the value of the property.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git commit -m "write commit message here": commits all staged work (It is important to write a brief, clear commit message so you know what each commit is for!)
  • git pull: pulls down any changes from your remote
  • git push: pushes your local changes up to your remote
  • git branch: shows you all your local branches and indicates which branch you are currently on
  • git merge name-of-branch: merges the specified branch into the branch you are currently on
  • git diff: shows you the changes in your unstaged code
